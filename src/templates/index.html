Bug Triaging

Data Science Project

In large software projects, hundreds of bug reports can be created daily, many of them containing incoplete, wrong or duplicate information. This project introduces techniques to decrease the amount of duplciates and to automatically assign priority and severity.

GitHub Repo

Why?

Explain why this is useful.

"Assigning severity to bug reports is highly dependent on the expertise of the bug reporters. Novice bug reporter might find it difficult to decide the right severity level."

Tian, Lo and Sun Information retrieval for fine-grained bug severity prediction... [1]

"Correct prioritization of bugs helps in bug fix scheduling/assignment and resource allocation. Failure of this will result in delay of resolving important bugs."

Sharma, Bedi, Chaturvedi and Singh Predicting the priority of a reported bug... [2]

"Users can, and often do, submit reports describing the same problem. The result is an added workload on the software maintenance engineers who have to ensure that each report is assigned a correct status."

Banerjee, Cukic and Adjeroh Automated duplicate bug report classification... [3]
